# MVP 范围冻结（当前迭代）

## 目标
在当前迭代内，只交付“文件稳定上传”最小闭环，不扩展平台化能力

## 范围分级

### A 类（本迭代必须完成）
- 文件监听与新增子目录发现
- 写入完成判定（静默窗口）与临时文件过滤
- 后缀过滤（可空）
- S3 上传主链路（队列 + worker + 重试）
- 失败可见（日志/状态）与手动补传
- 路径安全校验（禁止越权路径）
- 最小安全能力（接口鉴权 + CORS 白名单）
- 基线验证（后端测试 + 前端构建门禁）

### B 类（有价值，但本迭代不做）
- 告警规则增强
- 前端可视化细节优化
- 通知渠道拓展
- 运行时配置体验优化

### C 类（当前阶段明确不做）
- AI 日志总结与 AI 告警分析增强
- 多 Agent 管理与配置中心
- 路由编排与工作流能力
- 元数据落库、报表、SLA 体系
- 系统资源高级运维能力扩展

## 范围冻结规则
- 本迭代仅接受 A 类工作项
- 新需求默认进入 backlog，不插入当前冲刺
- 任何新增模块需说明如何直接提升“上传成功率/可恢复性/安全性”
- 若不能直接证明收益，则延期

## Done 统一口径
- 代码可构建
- 后端 `go test ./...` 通过
- 前端 `npm run build` 通过
- 文档同步更新（配置项、接口行为、默认值）
