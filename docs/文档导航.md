> 目标：把文档按用途分组，降低“找不到该看哪篇”的成本

## 建议阅读顺序

1. 先看：`docs/01-总览规划/平台概述.md`
2. 再看：`docs/01-总览规划/下一阶段执行计划.md`

4. 运行排障：`docs/02-开发运维/常见问题.md`
5. 质量验收：`docs/05-指标与评估/监控指标字典-v1.md`
