# 目录职责边界

## 1. 仓库级目录边界（GWF）

| 目录 | 职责 | 包含 | 不包含 |
| --- | --- | --- | --- |
| `go-watch-file/` | 后端执行平面与 API 服务 | Agent 运行、文件采集适配、告警决策、系统资源采集、知识库服务、控制面 API、运维脚本 | 前端页面源码、业务需求文档 |
| `console-frontend/` | 统一控制台前端 | 控制台页面、交互逻辑、API 契约类型 | 后端业务逻辑、采集执行脚本 |
| `docs/` | 平台文档与手册 | 定位、架构、模块说明、Roadmap、运维手册、评估模板 | 运行时数据、二进制产物 |
| `data/` | 运行与样例数据目录 | 本地数据、测试数据、示例输入输出 | 规范文档、源代码 |
| `reports/` | 阶段执行输出 | 回放结果、压测复盘、阶段报告原始产物 | 代码实现、长期规范文档 |
| `legacy/` | 历史归档 | 已下线或仅参考的历史方案 | 当前主线功能设计与运行手册 |

## 2. 文档目录边界（docs）

| 目录 | 职责 | 维护策略 |
| --- | --- | --- |
| `docs/00-文档治理/` | 文档体系治理规则与边界矩阵 | 任何文档结构调整先更新该目录 |
| `docs/01-平台定位/` | 项目定位、主线能力、范围边界 | 每次能力升级必更 |
| `docs/02-架构设计/` | 总体架构、关键数据流、技术约束 | 架构变更时同步更新 |
| `docs/03-能力模块/` | 功能模块职责、输入输出、接口边界 | 新模块上线或模块下线时更新 |
| `docs/04-路线图与计划/` | Roadmap 与迭代 TODO | 每次迭代启动/收尾更新 |
| `docs/02-开发运维/` | 开发与运行专题手册 | 仅写操作方法，不重复写平台定位 |
| `docs/03-告警与AI/` | 告警与 AI 专题手册及样本 | 仅写该域操作细节与验收 |
| `docs/04-知识库/` | 知识库专题文档和样本资产 | 仅写知识库域流程与评估素材 |
| `docs/05-指标与评估/` | 指标字典、阈值策略、回归报告 | 指标与报告持续累积 |
| `docs/98-历史兼容/` | 新旧文档映射、迁移说明 | 文档重构时维护映射 |
| `docs/99-架构附录/` | 数据结构图与流程图附录 | 作为架构说明的补充材料 |

## 3. 子目录文档边界约束

- `go-watch-file/README.md` 只描述后端服务定位、能力边界、API 与运行方式。
- `console-frontend/README.md` 只描述控制台定位、视图能力、前端运行与对接接口。
- `docs/` 主线文档负责“定义平台是什么”；专题手册负责“如何操作某个能力”。
- `reports/` 中的报告不反向定义能力边界，能力边界只能在 `docs/01` 与 `docs/03` 定义。
