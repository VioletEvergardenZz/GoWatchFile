# Roadmap（2026）

> 文档状态：现行（主线）  
> 校准日期：2026-02-22
> 口径说明：该文档为主线口径，发生边界变更时需优先更新。

## 1. 规划原则

- 以“当前真实能力”推进，不以历史叙事驱动优先级。
- 核心能力优先于辅助能力。
- 每个阶段必须有明确可验收产出（接口、指标、运行手册）。

## 2. 阶段路线

### 阶段 A（当前到下个 1~2 个迭代）

目标：稳定核心闭环，清理历史叙事偏差。

- 控制面：补齐任务生命周期可观测性和失败处理一致性。
- 告警：提升规则治理稳定性与抑制/升级解释性。
- 知识库：稳定引用式问答与推荐链路。
- 系统资源：完善主机视图可用性与权限边界。
- 文档：完成主线文档重构与目录职责收敛。

### 阶段 B（随后 2~4 个迭代）

目标：从“可运行”提升到“可运维”。

- 控制面：提升 Agent 分组治理与任务分发策略能力。
- 告警：支持更细粒度策略模板与异常分层。
- 知识库：完善审核流、回滚治理和运营指标闭环。
- 指标：统一 SLO 口径和阶段门禁。

### 阶段 C（中期）

目标：走向平台化扩展能力。

- 多输入源适配策略（文件之外的事件输入）。
- 更完整的策略编排与任务路由。
- 安全治理升级（权限分层、审计深度、密钥治理）。
- 跨环境发布与容量治理体系。

## 3. 核心能力与辅助能力的迭代预算

- 核心能力（控制面/告警/资源/知识库/控制台）：约 80%
- 辅助能力（文件适配、通知、AI 增强体验）：约 20%

## 4. 里程碑验收标准

每个里程碑至少满足：

1. 关键接口可回归。
2. 关键指标可观测。
3. 操作手册可执行。
4. 回滚路径明确。

## 5. 非目标声明（当前阶段）

- 不以“新增输入源数量”作为主成功指标。
- 不以“页面数量增长”作为平台化完成标准。
- 不引入脱离当前运维场景的复杂编排系统。
