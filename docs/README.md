## 2. 为什么会出现同序号目录

1. `docs/` 采用“双轨目录”策略：主线目录与专题兼容目录并存。
2. 主线目录是单一事实源：`01-平台定位`、`02-架构设计`、`03-能力模块`、`04-路线图与计划`。
3. 专题兼容目录保留既有脚本和历史链接：`02-开发运维`、`03-告警与AI`、`04-知识库`、`05-指标与评估`。
4. 因此 `02/03/04` 出现同序号不同主题属于兼容设计，不是重复建设。

## 4. 如何判断“最新文档”

1. 先查：`docs/00-文档治理/文档现行状态清单.md`。
2. 平台定位、架构、模块、Roadmap/TODO 一律以主线目录版本为准。
3. 专题目录不再重复定义主线边界，只补充运行细节、样本和验收。
4. 若主线与专题描述冲突，以主线目录为准并回补专题文档引用。

## 5. 维护原则

1. 每次能力边界调整时，至少同步更新：
   - `docs/01-平台定位/项目定位与主线能力.md`
   - `docs/02-架构设计/总体架构说明.md`
   - `docs/03-能力模块/功能模块说明.md`
   - `docs/04-路线图与计划/TODO.md`
2. 专题文档新增或更新后，必须回链对应主线文档。
3. 目录结构调整前，先更新治理目录：`docs/00-文档治理/`。
