## 2. 文档分层

- `docs/00-文档治理/`：文档地图、目录职责边界、维护规则。
- `docs/01-平台定位/`：项目定位、主线能力、范围边界。
- `docs/02-架构设计/`：总体架构、运行闭环、技术约束。
- `docs/03-能力模块/`：模块职责、输入输出、接口边界。
- `docs/04-路线图与计划/`：Roadmap 与迭代 TODO。
- `docs/02-开发运维/`、`docs/03-告警与AI/`、`docs/04-知识库/`、`docs/05-指标与评估/`：专题运行手册与评估资产（兼容保留）。
- `docs/99-架构附录/`：结构体映射与流程图等附录材料。
- `docs/98-历史兼容/`：旧文档到新体系的映射关系。

## 3. 维护原则

- 主线能力相关描述以 `01/02/03/04` 四层文档为准。
- 专题手册应引用主线文档，不重复维护定位与边界定义。
- 每次能力边界调整时，至少同步更新：
  - `docs/01-平台定位/项目定位与主线能力.md`
  - `docs/02-架构设计/总体架构说明.md`
  - `docs/03-能力模块/功能模块说明.md`
  - `docs/04-路线图与计划/TODO.md`
