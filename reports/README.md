# 阶段输出归档

> 文档状态：阶段输出入口
> 校准日期：2026-02-24
> 口径说明：本目录仅存放回放、压测、复盘等阶段产物，不反向定义现行平台主线与能力边界。
> 若与主线口径冲突，以 `docs/01-平台定位/项目定位与主线能力.md` 与 `docs/03-能力模块/功能模块说明.md` 为准。

## 目录用途

- 存放阶段执行输出，例如：压测结果、回放结果、阶段回归报告、复盘 JSON。
- 保持脚本输入输出路径稳定，避免自动化回归过程中的路径漂移。
- 便于按阶段做追溯，不承担平台定位、架构边界和模块职责定义。

## 命名建议

- 建议按“主题 + 日期”命名，例如：`stage-recap-2026-02-24.json`、`upload-recap-2026-02-24.md`。
- 对同一主题多次执行，可追加序号或时间戳，避免覆盖历史结果。
